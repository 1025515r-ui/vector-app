# VECTOR — Архітектурні рішення

## Ролі
- Тільки admin і teacher
- Батьків немає в системі

## Користувач (users в БД)
- email
- password_hash
- full_name
- role
- user_number (VEC-XXXXX випадковий)
- Все інше (телефон, заклад, місто, оплата) — в окремій CRM

## Адмін керує
- Додати педагога (email + ПІБ + роль)
- Видалити педагога
- Скинути пароль
- Активувати/деактивувати акаунт

## Педагог керує
- Своїми учнями
- Протоколами оцінювання
- Бачить аналітику своєї групи

## Рекомендації
- Фіксована база текстів
- Показуються коли навичка нижче середнього по групі
- Два варіанти: для педагога і для батьків (у звіті)
- Тексти вносяться пізніше в процесі роботи

## Звіти
- Тип 1: для педагога
- Тип 2: для батьків (документ, не роль)

## Домени
- vpid.org — EN лендінг
- ua.vpid.org — UA лендінг
- app.vpid.org — застосунок
- Піддомени для майбутніх серверів у різних країнах

## Адмін панель — два блоки

### Блок 1: Керування користувачами
- Синхронізація з платіжною системою (автоматично)
- Звіти для звірки з платіжною системою
- Додати/видалити педагога
- Активувати/деактивувати акаунт

### Блок 2: Статистика (прихована, тільки для власника)
- Кількість користувачів і динаміка
- Якість використання застосунку (активність)
- Кількість дітей в системі
- Наскрізна статистика: по групах, дітях, педагогах
- Для наукових досліджень

### Рекомендації
- Редагуються рідко, через програмування або через адмінку
- Не окремий блок адмінки

## ПРІОРИТЕТ: Повний річний звіт групи
- Найвищий пріоритет розробки
- Звіт по всій групі за навчальний рік
- Формати: PDF та Excel
- Включає: всі діти, всі 5 днів, динаміка, рекомендації
- Існуючий звіт в додатку — тільки для окремої дитини, не групи
